## 1. 프로젝트 개요

### 1.1 프로젝트명
AI 영수증 지출 관리 시스템 (Receipt Intelligence & Expense Tracker)

### 1.2 목적
영수증(이미지 또는 PDF)을 업로드하면 AI(Upstage Vision LLM)가 자동으로 내용을 인식하고, 지출 항목을 구조화하여 사용자의 소비 패턴을 손쉽게 추적·분석할 수 있는 웹 애플리케이션을 개발한다.

### 1.3 배경 및 필요성
- 수기로 가계부를 작성하는 방식은 번거롭고 지속하기 어렵다.
- 기존 가계부 앱은 직접 입력 방식으로, 사용자의 피로감이 높다.
- 영수증을 사진으로 찍거나 PDF로 저장하는 습관은 이미 보편화되었다.
- AI OCR 기술을 활용하면 영수증에서 자동으로 항목을 추출하여 입력 부담을 제거할 수 있다.

### 1.4 기대 효과

| 구분 | 기대 효과 |
|------|-----------|
| 사용자 편의성 | 영수증 촬영만으로 지출이 자동 기록되어 수기 입력 불필요 |
| 데이터 정확성 | AI 기반 OCR로 수기 입력 오류 최소화 |
| 소비 분석 | 카테고리별·기간별 통계 차트로 소비 패턴 파악 |
| 데이터 영속성 | SQLite DB에 저장하여 언제든 재조회 가능 |

## 2. 시스템 아키텍처

### 2.1 전체 흐름도

```
┌─────────────────────────────────────────────────────────┐
│                    사용자 (Browser)                      │
│              ReactJS + Vite + TailwindCSS               │
└───────────────────────┬─────────────────────────────────┘
                        │  HTTP (Axios)
                        ▼
┌─────────────────────────────────────────────────────────┐
│               백엔드 서버 (FastAPI)                      │
│                                                         │
│  [영수증 업로드 API]          [지출 조회 API]             │
│         │                          │                    │
│         ▼                          ▼                    │
│  [LangChain + Upstage]       [SQLite DB]               │
│   Vision LLM (OCR)           지출 내역 저장/조회         │
│   JSON 구조화 출력                                       │
└─────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────┐
│                    배포 인프라                           │
│           Git Repository  →  Vercel (CI/CD)            │
└─────────────────────────────────────────────────────────┘
```

### 2.2 핵심 데이터 흐름

| 단계 | 구간 | 기술 | 처리 내용 |
|:----:|------|------|-----------|
| 1 | 사용자 → 프론트엔드 | React + Axios | 영수증 이미지/PDF 파일 선택 및 업로드 요청 |
| 2 | 프론트엔드 → 백엔드 | FastAPI | multipart/form-data로 파일 전송 |
| 3 | 백엔드 → LLM | LangChain + Upstage | Vision LLM으로 이미지 분석, 구조화 JSON 반환 |
| 4 | 백엔드 → DB | SQLite | 추출된 지출 항목 저장 |
| 5 | DB → 프론트엔드 | FastAPI + Recharts | 저장된 지출 데이터 조회 및 차트 시각화 |

## 4. 주요 기능 명세

### 4.1 영수증 업로드 및 OCR 분석

| 구분 | 내용 |
|------|------|
| 입력 | 영수증 이미지(JPG, PNG) 또는 PDF 파일 |
| 처리 | Upstage Vision LLM이 이미지를 분석하여 날짜, 상호명, 항목명, 금액, 합계 추출 |
| 출력 | 구조화된 JSON 형태로 지출 항목 반환 후 DB 저장 |

OCR 출력 JSON 예시:
```json
{
  "date": "2025-04-10",
  "store_name": "이마트 강남점",
  "items": [
    { "name": "우유 1L", "quantity": 2, "price": 3200 },
    { "name": "식빵",   "quantity": 1, "price": 2500 }
  ],
  "total": 8900,
  "category": "식료품"
}
```

### 4.2 지출 내역 저장 및 조회
- 처리된 JSON 데이터를 SQLite DB에 자동 저장
- 날짜 범위, 카테고리, 상호명 기준 필터 조회 지원
- 페이지네이션으로 대량 데이터 처리

### 4.3 지출 통계 시각화
- 월별 지출 합계 막대 차트 (Recharts BarChart)
- 카테고리별 지출 비율 파이 차트 (Recharts PieChart)
- 일별 지출 추이 선 그래프 (Recharts LineChart)

### 4.4 영수증 이미지 관리
- 업로드된 원본 이미지 썸네일 표시
- 이미지 클릭 시 원본 확대 조회
- AI 추출 결과 수동 수정 기능

## 5. 데이터 모델 (SQLite)

### 5.1 receipts (영수증 테이블)

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | INTEGER | PK, AUTOINCREMENT | 영수증 고유 ID |
| store_name | TEXT | NOT NULL | 상호명 |
| date | DATE | NOT NULL | 구매 날짜 |
| total_amount | REAL | NOT NULL | 합계 금액 |
| category | TEXT | | 지출 카테고리 (식료품, 외식, 쇼핑 등) |
| image_path | TEXT | | 원본 이미지 저장 경로 |
| raw_json | TEXT | | LLM 출력 원본 JSON |
| created_at | DATETIME | DEFAULT NOW | 레코드 생성 시각 |

### 5.2 receipt_items (지출 항목 테이블)

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | INTEGER | PK, AUTOINCREMENT | 항목 고유 ID |
| receipt_id | INTEGER | FK → receipts.id | 영수증 참조 ID |
| item_name | TEXT | NOT NULL | 상품명 |
| quantity | INTEGER | DEFAULT 1 | 수량 |
| unit_price | REAL | NOT NULL | 단가 |
| total_price | REAL | NOT NULL | 소계 (수량 × 단가) |

## 6. API 명세 (초안)

### 6.1 FastAPI 엔드포인트 목록

| 엔드포인트 | Method | URL | 설명 |
|-----------|--------|-----|------|
| 영수증 업로드 | `POST` | `/api/receipts/upload` | 이미지/PDF 업로드 및 OCR 분석 후 저장 |
| 영수증 목록 조회 | `GET` | `/api/receipts` | 전체 영수증 목록 조회 (필터/페이지 지원) |
| 영수증 상세 조회 | `GET` | `/api/receipts/{id}` | 특정 영수증 상세 정보 및 항목 조회 |
| 영수증 수정 | `PUT` | `/api/receipts/{id}` | 영수증 정보 및 항목 수동 수정 |
| 영수증 삭제 | `DELETE` | `/api/receipts/{id}` | 영수증 및 관련 항목 삭제 |
| 지출 통계 조회 | `GET` | `/api/stats/summary` | 기간별/카테고리별 지출 통계 데이터 반환 |
| 카테고리 목록 | `GET` | `/api/categories` | 사용 가능한 지출 카테고리 목록 반환 |

## 8. 배포 구성

### 8.1 Git 브랜치 전략

| 브랜치명 | 역할 |
|----------|------|
| `main` | 운영 배포 브랜치 — Vercel 자동 배포 트리거 |
| `develop` | 개발 통합 브랜치 — 기능 브랜치 병합 후 테스트 |
| `feature/*` | 기능 단위 개발 브랜치 (예: feature/receipt-upload) |
| `hotfix/*` | 운영 긴급 수정 브랜치 |

### 8.2 Vercel 배포 방식
- GitHub 저장소와 Vercel 프로젝트 연동
- `main` 브랜치 push 시 자동 빌드 및 배포 (CI/CD)
- Pull Request 생성 시 Preview URL 자동 생성 (QA 용이)
- 환경변수(`UPSTAGE_API_KEY` 등)는 Vercel Dashboard에서 관리

### 8.3 배포 파이프라인

```
개발자 로컬
    │  git push
    ▼
GitHub Repository (main 브랜치)
    │  Webhook 트리거
    ▼
Vercel CI/CD
    │  npm run build (Vite)
    ▼
Vercel CDN (글로벌 엣지 네트워크)
    │  정적 파일 서빙
    ▼
사용자 브라우저
```

## 10. 향후 확장 가능성

### 10.1 기능 확장 아이디어

| 확장 기능 | 설명 |
|-----------|------|
| AI 카테고리 자동 분류 | LLM이 상호명/항목명 기반으로 카테고리 자동 태깅 |
| 월별 예산 설정 및 알림 | 카테고리별 예산 설정 후 초과 시 알림 기능 |
| 다중 사용자 지원 | JWT 인증 기반 개인별 지출 데이터 분리 관리 |
| DB 마이그레이션 | SQLite → PostgreSQL로 전환하여 대규모 데이터 처리 |
| 모바일 앱 | React Native 또는 PWA로 모바일 카메라 직접 연동 |
| AI 소비 리포트 | LLM 기반 월별 소비 패턴 분석 리포트 자동 생성 |
| 가계부 내보내기 | Excel/CSV/PDF 형식으로 지출 내역 내보내기 |

### 10.2 기술 스택 업그레이드 경로
- **SQLite → PostgreSQL**: 사용자 증가 및 동시접속 대응
- **Vercel Functions → AWS Lambda / GCP Cloud Run**: 백엔드 서버리스 전환
- **LangChain → 커스텀 파이프라인**: 프롬프트 최적화 및 비용 절감
